## 2. バリュープロポジション

### pikuraでしか得られない価値

1. **情報の一元化**: ランキング・大会・体験会・コーチング・記事・ショップが1つのサイトに集約されている。競合はそれぞれバラバラ
2. **選手ページ**: JPA公式にはないデジタルな選手プロフィールページ。ランキング情報をSNSでシェアできる
3. **マルチソースのイベント集約**: JPA・テニスベア・PJFなど複数団体のイベントを横断検索できる唯一のサイト
4. **手数料ゼロのコーチングマッチング**: プラットフォーム手数料なし。選手・コーチにも、受講者にも優しい設計

### 競合との差別化

| | pikura | JPA公式サイト | テニスベア |
|---|---|---|---|
| ランキング | Web上で検索・閲覧・共有 | PDFのみ | なし |
| イベント | 全国250+件、マルチソース | 自団体のみ | コート予約メイン |
| 記事メディア | 20本+、入門～上級 | なし | なし |
| コーチング | 手数料ゼロ、5%OFF | なし | なし |
| ショップ | ピックルボール専用グッズ | なし | なし |
| パートナー探し | ペア募集（開発中） | なし | なし |

### ワンライナー（キャッチコピー）

**推奨案: 「ピックルボールするなら、ピクラ。」**

- 理由: 「何のサイトか」「何をすればいいか」が1秒で伝わる。動詞が入っているため能動的。「〜するなら〜」構文は日本語として自然で、記憶に残りやすい
- 代替案: 「大会も、情報も、仲間も。」（サブコピーとして使用可）

## 3. キーメッセージ体系

### ヘッドライン（H1）候補 3案

**A案（推奨）: 「ピックルボールするなら、ピクラ。」**
- 強み: シンプル。サービス名と用途が直結する。口コミで広がりやすい
- サブヘッドライン: 「大会情報、ランキング、コーチング、入門ガイド。日本のピックルボール情報がここに集まる。」

**B案: 「あなたのピックルボール、ここから始まる。」**
- 強み: 初心者にも経験者にも対応する。「始まる」には最初の一歩と新しいステージの両方の意味がある
- サブヘッドライン: 「250+のイベント情報、124名のランキング、プロによるコーチング。日本最大のピックルボールプラットフォーム。」

**C案: 「探すをやめよう。」**
- 強み: 課題（情報の分散）に直接訴求する。短くインパクトがある
- サブヘッドライン: 「大会、ランキング、コーチ、仲間。ピックルボールに必要な情報が、全部ここにある。」

### 各機能セクションの訴求コピー

#### イベント・大会情報
- **見出し**: 「全国250件以上のイベントを、まとめて検索」
- **本文**: 「JPA・テニスベア・PJFなど複数の団体が主催する大会・体験会・練習会をひとつの画面で。都道府県、レベル、日程で絞り込んで、あなたにぴったりのイベントが見つかる。」
- **CTA**: 「イベントを探す」

#### ランキング
- **見出し**: 「JPA公式ランキングを、いつでも確認」
- **本文**: 「124名の選手データをカテゴリ別に閲覧。自分の順位をチェックしたり、注目選手を見つけたり。選手ページをSNSでシェアすることもできる。」
- **CTA**: 「ランキングを見る」

#### コーチング
- **見出し**: 「プロから直接学べる。手数料ゼロ。」
- **本文**: 「経験豊富なコーチやプロ選手から、個人レッスン・グループレッスンを受けられる。ピクラ経由の申し込みでレッスン料5%OFF。掲載無料・手数料ゼロだからコーチにも安心。」
- **CTA**: 「コーチを探す」

#### 入門ガイド・記事
- **見出し**: 「ルールも戦術も、読めばわかる」
- **本文**: 「ピックルボールとは？から始まる入門ガイド、基本ルール、戦術テクニック、ギアレビューまで。初心者から大会出場を目指す人まで、レベルに合った記事が見つかる。」
- **CTA**: 「記事を読む」

#### ペア募集（準備中）
- **見出し**: 「パートナー探しも、もうすぐ。」
- **本文**: 「大会に出たいけどパートナーがいない。そんな悩みを解決するペアマッチング機能を開発中。リリースをお楽しみに。」
- **CTA**: なし（準備中表示）

#### ショップ

## 4. トーン & マナー

### ブランドの人格

**「親しみやすいスポーツ仲間」**

pikuraは先輩でもコーチでもなく、一緒にプレーする仲間。詳しいけど偉そうじゃない。頼れるけどカジュアル。

| 属性 | pikuraのトーン | やらないこと |
|------|-------------|-------------|
| フレンドリー | 「〜できます」「〜が見つかる」 | 「〜すべきです」「〜しなさい」 |
| 情報に正確 | 具体的な数字を出す（250件、124名） | 「たくさんの」「様々な」等の曖昧な表現 |
| 簡潔 | 一文は短く。箇条書きを多用 | 長い説明文、回りくどい表現 |
| プレイヤー目線 | 「あなたの」「探す」「見つける」 | 「弊社は」「サービスのご紹介」 |

### 言葉遣い

- **敬語レベル**: 丁寧語（です・ます）をベース。ヒーローコピーのみ体言止めOK
- **カタカナ**: ピックルボール用語はそのまま使う（ディンク、ドロップ、キッチン等）。IT用語は避ける（「プラットフォーム」は使わず「サイト」で十分）
- **一人称**: 使わない。「ピクラ」と呼ぶ場合は最小限
- **呼びかけ**: 「あなた」は控えめに。「〜したい人」「〜を探している人」で間接的に

### ビジュアルの方向性

- **色**: 現状のprimaryカラーを維持。スポーツらしい明るさと、情報サイトとしての読みやすさを両立
- **写真**: 可能であれば実際のプレー風景を使いたい（現状はイラスト/テクスチャのみ）
- **アイコン**: 機能セクションには絵文字ではなくシンプルなアウトラインアイコンを使用（Lucide Icons推奨、shadcn/uiと統一）
- **余白**: 広めに取る。情報密度を下げて初心者にも読みやすく
- **フォント**: 現状のままでOK。太字は見出しのみ

## 5. LP構成の提案

### 現状の問題点（再整理）

1. ヒーローのCTAが「ランキングを見る」一択 → ランキングに興味がない人（全体の8割以上）は「自分向けじゃない」と離脱
2. セクション順がランキング → 記事 → イベント → CTA。機能が羅列されているだけで、「このサイトで何ができるか」のストーリーがない
3. コーチング・ペア募集・ショップがトップページに一切出ていない
4. 数字の訴求が弱い（250件のイベント、20本の記事、etc.）
5. モバイルでボタンが小さく、CTAが目立たない

### 新LP構成（上から順に）

```
1. ヒーロー           ← 「何のサイトか」を1秒で伝える
2. 数字で信頼          ← 実績を3つの数字で示す
3. イベント・大会       ← 最もユーザーニーズが広い機能を最初に
4. ランキング          ← コア機能だが、2番目の位置に
5. コーチング          ← 差別化ポイント。手数料ゼロを強調
6. 記事メディア         ← 初心者の入口として
7. ショップ            ← 軽めに紹介
8. 最終CTA            ← 行動を促す
```

### 各セクション詳細

#### Section 1: ヒーロー

**役割**: ファーストビューで「何のサイトか」「誰向けか」「何ができるか」を伝える

**レイアウト**:
```
[Badge] 日本最大のピックルボール情報サイト

[H1] ピックルボールするなら、ピクラ。

[Sub] 大会情報、ランキング、コーチング、入門ガイド。
      日本のピックルボール情報が、ここに集まる。

[CTA Primary] イベントを探す
[CTA Secondary] 記事を読む
```

**変更点**:
- Badgeを「JPA公式ランキング公開中」→「日本最大のピックルボール情報サイト」に変更。初訪問者にサイトのポジションを伝える
- H1を変更。便益が1秒で伝わるコピーに
- Primary CTAを「ランキングを見る」→「イベントを探す」に変更。イベント検索はターゲット全員に刺さる行動
- Secondary CTAを「記事を読む」に維持（初心者向けの入口として有効）

**モバイル考慮**:
- CTAボタンは縦並び、幅100%
- サブテキストは2行以内に収める

## 実装上の注意点

### セクション順の根拠

| 優先度 | セクション | 根拠 |
|--------|-----------|------|
| 1 | イベント | ターゲット3層すべてに刺さる。「近くで大会/体験会がないか？」は最も普遍的なニーズ |
| 2 | ランキング | 競技志向ユーザーの主要導線。ただし初心者には刺さらないため2番目 |
| 3 | コーチング | 差別化要因。手数料ゼロは競合にない強み |
| 4 | 記事 | SEO流入のメイン。ただしLP上では行動喚起力が低いため後半 |
| 5 | ショップ | 収益化導線だが、現状の購買意欲は低い |

### CTA設計の原則

- **Primary CTA**: 「イベントを探す」に統一（全ターゲットに有効な行動）
- **Secondary CTA**: セクションごとの個別CTA（「ランキングを見る」「コーチを探す」「記事を読む」）
- **ボタンサイズ**: モバイルでは幅100%、最低44px高さ（タップ領域確保）
- **文言**: 「〜を見る」「〜を探す」「〜を読む」で統一。「〜はこちら」「詳細」は使わない

### モバイルファーストの設計方針

1. **ヒーロー**: 背景画像の代わりにグラデーションのみ（LCP改善）。テキストの視認性を最優先
2. **Stats**: 3カラムを維持するが、数字のフォントサイズを大きく、ラベルを小さく
3. **カードグリッド**: モバイルでは1カラム、タブレットで2カラム、PCで3カラム
4. **CTA間隔**: セクション間にCTAを挟む。スクロールのどの位置でもアクションできるように
5. **フッターCTA**: スクロール追従型は使わない（UXを損ねる）
