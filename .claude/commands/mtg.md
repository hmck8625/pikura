# 社内幹部MTG

pikura.app プロジェクトの幹部MTGを開催し、議事録を作成します。
Day終了時、Week振り返り時、緊急議題発生時などに使用。

## 手順

1. 現在のプロジェクト状態を収集する
   - `git log --oneline -20` で直近のコミット履歴
   - 当日の日報があれば読み込む（`week{N}/YYYY-MM-DD_日報.md`）
   - 前回のMTG議事録を読み込む（最新の `week{N}/*MTG*.md`）
2. KPI・数値データを更新する
   - 記事数（microCMS）、アイキャッチ画像数、YouTube動画数
   - イベント数、選手数、ショップ商品数
   - サイトマップURL数
   - GSC/YouTube Analyticsデータ（あれば）
3. 出席者を設定する
   - プロジェクトリード（人間）
   - AIアシスタント（Claude）
   - 大会運営ディレクター「河野」（AI人格）
4. 以下の構成で議事録を作成する

## 議事録テンプレート

```markdown
# 幹部MTG議事録 — YYYY-MM-DD {テーマ}

## 出席者
## 0. 本日のMTG目的
## 1. 実績 — 計画 vs 実績（表形式）
## 2. KPI更新（表形式: 前回値 → 今回値 → 変化）
## 3. 課題・ブロッカー
## 4. 河野の所見（引用形式で3-5段落）
## 5. 次のアクション（Tier 1/2/3 に分類）
## 6. スコアカード（プロダクト/コンテンツ/テクノロジー/マーケ/マネタイズ/コスト効率）
## 7. 決定事項（番号付きリスト）
```

5. 河野の所見を書く際のルール:
   - 率直・辛口。数字に基づく分析
   - 良い点は素直に褒める。問題点は遠慮なく指摘
   - 次に何をすべきかの具体的な提案を含める
   - 引用ブロック（`>`）で記述
6. 議事録を `/mnt/c/Users/daisu/Desktop/ピ/week{N}/` に保存
   - 命名: `YYYY-MM-DD_幹部MTG_{テーマ}.md`

## MTGの種類

| 種類 | タイミング | テーマ例 |
|------|-----------|---------|
| Day終了レビュー | 作業日の終わり | Day1終了、Day2終了 |
| Week振り返り | 週末 | Week9振り返り_Week10計画 |
| 中間レビュー | 週の中頃 | Week9中間レビュー |
| 緊急MTG | 随時 | SEO戦略変更、マネタイズ方針 |
